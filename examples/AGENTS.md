# AGENTS.md テンプレート

このファイルは Codex CLI がプロジェクトで作業する際の指示書です。
プロジェクトのルートディレクトリに配置してください。

## コーディング規約

- 言語: TypeScript
- スタイル: 関数型プログラミング推奨
- 命名規則: 変数・関数は camelCase、型・クラスは PascalCase
- インデント: スペース2つ
- 行の長さ: 最大100文字
- セミコロン: 必須

## ディレクトリ構成

```
src/
  components/    # UIコンポーネント
  hooks/         # カスタムフック
  utils/         # ユーティリティ関数
  types/         # 型定義
  services/      # API・外部サービス連携
tests/
  unit/          # ユニットテスト
  integration/   # 結合テスト
```

## 技術スタック

- ランタイム: Node.js 22+
- フレームワーク: （プロジェクトに合わせて記入）
- テスト: Vitest
- リンター: ESLint + Prettier

## テスト方針

- 新規関数には必ずユニットテストを書くこと
- テストファイル命名: `*.test.ts`
- 最低カバレッジ: 80%
- テスト名は具体的に: `it('Yが与えられたときXを返すべき')`

## 禁止事項

- TypeScript で `any` 型を使わないこと
- `var` 宣言を使わないこと
- `console.log` をコミットしないこと
- 本番コードで同期ファイル操作を使わないこと
- コードにシークレットを埋め込まないこと

## 注意事項

- 新しいパターンを作る前に、既存のパターンを必ず確認すること
- 継承より合成を優先すること
- 関数は小さく保つこと（最大30行）
- エラーは明示的に処理し、握りつぶさないこと
